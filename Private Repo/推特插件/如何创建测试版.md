# 📦 如何创建完整的测试版本

## 🎯 目标
将模块代码和 UserScript 头部组合成一个完整的可安装文件。

---

## 📝 步骤

### 方法一：手动组装（推荐）

1. **打开文件**:
   - `【测试版】Twitter-X 智能助手 v3.0.user.js`

2. **找到这一行**:
   ```javascript
   // [在这里粘贴 Twitter-X 智能助手模块 v3.0.js 的全部内容]
   ```

3. **复制模块代码**:
   - 打开 `Twitter-X 智能助手模块 v3.0.js`
   - 全选并复制（Ctrl+A, Ctrl+C）

4. **粘贴到测试版文件**:
   - 在 `【测试版】Twitter-X 智能助手 v3.0.user.js` 中
   - 找到上面提到的那一行
   - 删除那一行
   - 粘贴模块代码（Ctrl+V）

5. **保存文件**

6. **安装到 Tampermonkey**:
   - 打开 Tampermonkey 管理面板
   - 点击 "+" 创建新脚本
   - 删除默认内容
   - 粘贴完整的测试版代码
   - 保存（Ctrl+S）

7. **测试**:
   - 访问 https://x.com
   - 打开控制台（F12）
   - 应该看到初始化成功的消息

---

### 方法二：使用 PowerShell 自动组装

在推特插件文件夹中执行：

```powershell
# 读取模块代码
$moduleCode = Get-Content "Twitter-X 智能助手模块 v3.0.js" -Raw -Encoding UTF8

# 读取测试版框架
$testFramework = Get-Content "【测试版】Twitter-X 智能助手 v3.0.user.js" -Raw -Encoding UTF8

# 替换占位符
$completeCode = $testFramework -replace '\[在这里粘贴 Twitter-X 智能助手模块 v3.0.js 的全部内容\]', $moduleCode

# 保存完整版本
$completeCode | Set-Content "【完整测试版】Twitter-X 智能助手 v3.0.user.js" -Encoding UTF8

Write-Host "✅ 完整测试版本已创建！" -ForegroundColor Green
```

## ⚠️ 注意事项

1. **编码问题**: 确保文件保存为 UTF-8 编码
2. **完整性**: 确保模块代码完整复制，没有遗漏
3. **测试环境**: 建议在测试账号上测试
4. **速率限制**: 不要设置过高的自动化频率

---

## 🐛 常见问题

### Q: 控制台显示 "TwitterAssistant 对象未找到"
**A**: 模块代码没有正确粘贴，请检查步骤 3-4

### Q: 初始化失败
**A**: 检查控制台错误信息，可能是代码不完整

### Q: 自动化不工作
**A**: 确保已启用功能开关并调用 start() 方法
